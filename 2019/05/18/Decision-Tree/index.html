Decision-Tree&Boosting

Posted on 2019-05-18 | In data analysis
The note for Dicision Tree and Boosting.

Decision Tree

continuous feature

finding error with small error on the training set

  1. start with the root node
  2. at each step split one of the leaves
  3. reqeat until a termination criterion

spliting rule

to minimize entropy

split on percentile for continuous features

It is computationally expensive to sort by feature in large datasets. If we have datasets that are extremely large, we would have to sort according to feature for each datapoint. Instead, we want to split with percentiles.

pruning trees

to reduce overfitting

Fully-grown trees (trees where all leaves are pure) overfit the data because its training error is 0. To avoid this overfitting, we discussed two methods: pruning and bagging.

bagging

to reduce the variance of trees

predict using the majority vote over the trees

Each tree is given a randomly selected asset of the training data. This is good way to improve performance and avoid overfitting data. Because each tree has a different asset of data and casts its own vote, we use the majority vote to make the final prediction.

generative and discriminative modeling

generative

two split separately

to explain how data is generated

Goal: Probabilities estimates

Performance measure: likelihood

Mismatch problems: outliers

Generative models choose an optimal threshold based simply on probability distributions.

discriminative

all data together

to predict property/label of the data

Goal: classification rule

Performance measure: misclassification rate

Mismatch problems: misclassifications

Discriminative models are better for dealing with outliers because they do not look at any probability distributions. Rather they take a threshold with the smallest number of mistakes. It’s more robust when there are many outliers.

Boosting

In the boosting process,

  1. The weak learner generates a weak rule from the weighted training set
  2. We change the weights of the examples based on the weak rule
  3. Repeat steps 1-2 until we have many weak rules
  4. We make a strong rule, the weighted average of weighted sum of all the weak rules.

With boosting, our test error continues to decrease even if we have a training error of 0.

In regular machine learning, we expect the test error to shoot up once we have a 0 training error because we have overfit the training data. With boosting, our test error continues to go down. So, even if we have many parameters (even over 2,000,000 for just 10,000 training examples), our prediction capability is not restricted!

Weight decay does not decrease bias. Recall that weight decay is decreasing the magnitude of parameters, so it increases bias.

Bagging decreases data variation without significantly increasing model bias.

Boosting reduces both variation and model bias.

Adaboost

Remember we have many weak learners that combine to form a strong learner that makes the decisions. Adaboost is an algorithm for combining the weak learners.

Margins measure the correctness of a prediction.

Application of boosting: face detection

find best weak rules using exhaustive searching

The work goes into designing informative features.

Boosting works well when the classifier is depending on a small subset of features.

The algorithm had to learn using labelled data, taking squares which either contained or did not contain a face. This is an example of supervised learning.

Instead of computing all the features to decide if it is a face or not, it computes in stages. We calculate stage by stage, and only continue to the next stage if the face (or other object you are recognizing) isn’t rejected. As we continue down the cascade, we increase likelihood of the box to be the item.

Ensembles

Decision trees have high data variation.

Random forest

Based on bagging trees

All trees can be learned in parallel.

With random forests, each tree will be more different and increase the power of averaging. The higher variation allows you to search over fewer things, speeding up the learning process. Random forests are also based on bagging, or “Bootstrap Aggregation”.

Gradient Tree Boosting

The trees are trained sequentially, one after the other.

Each is trained on a weighted training set.

Boosted gradient trees are based on the sins of their forefathers, and so receive a different weight for each tree. Random forests, on the other hand, are learned in parallel and trained similarly, and so get no special treatment.

classification of XGBoost

False Positive - giving unnecessary treatment - more medical bills, but still alive.

False Negative - withholding necessary treatment - death!

The ROC curve is given by the proportion of: True Positive rate / False Positive rate
