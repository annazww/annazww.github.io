A review for operating system.

Memory Management

Theory

Allocation of memory occurs when new process is created or process requests more memory.

Freeing of memory occurs when process exits or process no longer needs memory it requested.

If process given CPU->time, must also be in memory->space.

Context switch only between processes in memory.

After memory is released, it will coalesce with adjacent.

Selecting the Best Hole

First (or next) fit: Simple | Fast | creates smaller holes, might never be used because hole is too small

Best fit: Must check every hole | Leaves very small fragments | may be optimal | average hole size is small

Worst fit: Must check every hole | Leaves large fragments | may be optimal | leave the largest hole every time to be reused again

Memory is cheap, but time is expensive.

Fragmentation

Fragmentation is caused by variable-size allocations.

Lots of little pieces of holes in memory that are useless.

Internal fragmentation: Unused space within (allocated) block | Cannot be allocated to others | Can come in handy for growth

External fragmentation: Unused space outside any blocks (holes) | Can be allocated

What if No Holes

Compaction: Simple idea | very time consuming

Break block into sub-blocks: Easier to fit | complex

Consider time vs. complexity tradeoff: time is more important.

50% Rule

m = n/2: n = number of blocks | m = number of holes = n/2 | correct if all allocations are imperfect fits

Unused Memory Rule

f = k/(k+2): k is ratio of avg hole size to avg block size | f is fraction space lost to holes | k → ∞, f → 1 | k → 0, f → 0

Logical Memory

Theory

Logical memory = a process’s memory

Allocated without regard to physical memory.

Memory Registers Part of Context

Benefit: Allows each process to be separately located | Protects each process from all others

Segmented Address Space

Segment-based Address Translation: physical address = ST (s) + i

Pro: Each segment can be located independently | separately protected | grown/shrunk independently

Pro: Segments can be shared by processes.

Con: Variable-size allocation | Difficult to find holes in physical memory | External fragmentation

Paged Address Space

Pages and frames: Frame: a physical unit of information | A page fits exactly into a frame | Fixed size, all pages/frames same size

Page-based Address Translation: physical address = PT (p) || i

Page table “maps” logical page into frame of physical memory.

Segments vs. Pages

Segment is good “logical” unit of information. Can be sized to fit any contents | Makes sense to share (e.g., code, data) | Can be protected according to contents

Page is good “physical” unit of information. Simple memory management

Combining Segments and Pages

Address Translation: Do various checks | Use s to index segment table to get page table | Use p to index page table to get frame f | Physical address = concatenate (f, i)

Run slower than a pure one.

TLB: Translation Look-aside Buffer

Locality: Fast memory keeps most recent translations.

The larger the TLB, the higher the hit rate | the slower the response | the greater the expense.

Put the most frequently translated page numbers.

TLB must be flushed on context switches. Different processes have different page table.

Virtual Memory

Theory

Virtual memory: illusion of large memory. Keep only portion of logical memory in physical | Rest is kept on disk (larger, slower, cheaper) | Unit of memory is segment or page (or both)

Page Faults are Expensive. But if very rare, tolerable.

Principle of Locality: Make sure most referenced pieces in memory.

References cluster in time/space.

Basic Page Replacement Algorithms

• FIFO: select page that is oldest – Simple: use frame ordering
– Doesn’t perform very well (oldest may be popular)

• OPT: select page to be used furthest in future – Optimal, but requires future knowledge – Establishes best case, good for comparisons – not realistic

• LRU: select page that was least recently used – Predict future based on past; works given locality – Costly: time-stamp pages each access, find least

Approximating LRU: Clock Algorithm

Hardware support: reference bit

How reference bit is used:
– When frame filled with page, set bit to 0 (by OS)
– If frame is accessed, set bit to 1 (by hardware)

Resident Set Management

• Local: limit frame selection to reques6ng process – Isolates effects of page behavior on processes

– Inefficient: some processes have unused frames • Global: select any frame (from any process)

– Efficient: resident sets grow/shrink accordingly
– No isola6on: process can negatively affect another

Multiprogramming Level: Resident set should contain the working set.

Denning’s Working Set Model: Add/remove pages according to W(t, ∆).

Working set is a local replacement policy – Process’s page fault behavior doesn’t affect others.

Clock VS Denning’s Working Set Model

– Clock: simple, easy to implement, global policy

– Denning’s Working Set Model: difficult to implement, but process’s page fault behavior doesn’t affect others.

Contrast

OPT ≥ LRU ≳ Clock ≳ FIFO
