No.39 Combination Sum

https://leetcode.com/problems/combination-sum/

analysis

Backtracking is useful in combination problem. The core idea is appending element to the memorization and popping it out after finishing one depth first search process. If finding one possible combination, add it into the result. The running time may be $O(2^n)​$ in worst case.

For running time, a simple inference is $T(n)=T(1)+T(2)+T(3)+…+T(n-1)​$. So $T(n+1)=(T(1)+T(2)+T(3)+…+T(n-1))+T(n)=2T(n)​$. As soon as n is added $1​$ more, the running time may be doubled. Therefore, the running time would be $O(2^n)​$.

No.40 Combination Sum II

https://leetcode.com/problems/combination-sum-ii/

analysis

The difference with No.39 is that every number can be used only once and no duplication. So we need to sort the $candidates$ at first. Also, we need to add a visited array to jusge whether we need to add the number into memorization. It is for the continuous identical numbers. If the previous number has been visited, we can pass it without any implementation, which saves time. The running time is $O(2^n)$.

No.377 Combination Sum IV

https://leetcode.com/problems/combination-sum-iv/

analysis

DP is easier to discuss about for this problem. We define DP formula as $DP[i]$, which represents the number of combination for $sum=i$. And for $DP[i+1]$, imagine we only need to add one more to reach target, this number can be any one in the array. So we can get $DP[i] = DP[1]+DP[2]+DP[3]+…+DP[i-1]$. The running time would be $O(n^2)$ for two loops.

Follow up

What if negative numbers are allowed in the given array? Infinite loop.
How does it change the problem? $if$ $num<=target-sum$ may be deleted.
What limitation we need to add to the question to allow negative numbers? Make a restriction on the length of subset.
