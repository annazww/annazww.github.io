No.560 Subarray Sum Equals K

https://leetcode.com/problems/subarray-sum-equals-k/

analysis

The code idea of this problem is that if one prefix sum minus another prefix sum is equal to target $k$, the subarray between these two points is a target subarray and we don’t need to remember what it is, just counting the number. So we need to traverse the array and memorize every prefix sum, plus their occurance number. The running time is $O(n)$.

No.974 Subarray Sums Divisible by K

https://leetcode.com/problems/subarray-sums-divisible-by-k/

analysis

We memorize the residual of every prefix sum divided by $K$. If we find two identical residuals, the subarray betwwen these two points is the target subarray. The running time is $O(n)$.

code

No.698 Partition to K Equal Sum Subsets

https://leetcode.com/problems/partition-to-k-equal-sum-subsets/

analysis

It’s a typical combination problem. We use backtracking to solve it. There are three cases in backtracking process. If $k=1$, it is exactly true that the whole array is the target subarray. If $sum=target$, we go recursion and aim to find $k-1$ in rest array. Otherwise, we traverse every possible element and do depth first search.

The running time should be $O(n)$.

code

No.930 Binary Subarrays With Sum

https://leetcode.com/problems/binary-subarrays-with-sum/

analysis

We can use hashtable and prefix sum to solve this problem. Here we talk about another method. We memorize the number of $0s$ between every $1$ and $1$ pair. And traverse the memorization and update the result number. The running time is $O(n)​$ as well.

code

No.523 Continuous Subarray Sum

https://leetcode.com/problems/continuous-subarray-sum/

analysis

We use hashtable and prefix sum to solve this problem. We need to discuss seperately for $k=0$ and $k=1$. We memorize the first index of different prefix sum we have encountered during traversal. Then check whether the subarray contains no less than 2 elements. The running time should be $O(n)$.
