No.647 Palindromic Substrings

https://leetcode.com/problems/palindromic-substrings/

analysis

We traverse every element of input $s$ and treat it as the $middle$ of palindromic string to check every possible string. There are two cases. One is odd case, and the other one is even case. The running time is $O(n^2)$.

No.855 Exam Room

https://leetcode.com/problems/exam-room/

analysis

We use heap to solve this problem. We use $maxheap$ here to remember the distance between every assigned seat pair. Because if we find the largest distance, we can assign a new seat between them to achieve the largest distance. The $pop$ and push implementation need $O(logn)$. So the running time for $seat$ and $leave$ should be both $O(long)$.

code

No.923 3Sum With Multiplicity

https://leetcode.com/problems/3sum-with-multiplicity/

analysis

This is a typical permutation and combination problem. There are three main cases. The first one is that the sum of three equal number is target. The second one is that two numbers are the same and one different. The third one is for three different numbers. Importantly, we have to define exact condition for these three cases and introduce a memo array to remember the combination we have found to avoid duplicate counting. The time would be $O(n^3)$, where n is related to the number of different numbers in the input.

code

No.3 Longest Substring Without Repeating Characters

https://leetcode.com/problems/longest-substring-without-repeating-characters/

analysis

Keep a pointer to iterate the every character in the string and a queue to memorize the characters that we have already seen. When we have ever seen a character, we pop the element out from the the beginning of the queue until the identical character is not existing in the queue. Always update the queue during iteration.

The running time is $O(n)$.

code

No.5 Longest Palindromic Substring

https://leetcode.com/problems/longest-palindromic-substring/

analysis

We can iterate every character in the string as the middle of a substring and check the longest palindrom we can reach. We can inplement in two situations: one is with odd number of characters and the other one is with even number of characters.

The running time is $O(n^2)$.
