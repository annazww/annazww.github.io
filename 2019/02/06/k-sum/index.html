No.653 Two Sum IV - Input is a BST

https://leetcode.com/problems/two-sum-iv-input-is-a-bst/

analysis

Preorder traversal in binary search tree can return the value of node in increasing order. Then we use two pointers from the beginning and the end to judge whether there are two digits whose sum is the target. The running time is $O(n)â€‹$.

No.167 Two Sum II - Input array is sorted

https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/

analysis

Since the array has already been sorted, we use two pointers to traverse the array. The running time is $O(n)$.

code

No.170 Two Sum III - Data structure design

https://leetcode.com/problems/two-sum-iii-data-structure-design/

analysis

A simple but easy idea is using hashtable. $Find$ implementation is almost the same as No.1 Two Sum. The running time for $add$ is $O(1)$. The running time for $find$ is $O(n)$.

code

No.15 3Sum

https://leetcode.com/problems/3sum/

analysis

Fix one number and simplify the problem into No.1 Two Sum. The running time is $O(nlogn)$.

code

No.16 3Sum Closest

https://leetcode.com/problems/3sum-closest/

analysis

Update the minimum gap between target and the sum of 3 numbers. Then the problem can be nearly simplified to No.15 3Sum. The running time is $O(nlogn)$.

code

No.259 3Sum Smaller

https://leetcode.com/problems/3sum-smaller/

analysis

The same idea as No.15 3Sum. But we cannot use binary search here. So the running time is $O(n^2)$.
