A leetcode collection for interval related problems. Greedy algorithm is common to use in these problems. Priority queue is a good data structure for them.

No.56 Merge Intervals

https://leetcode.com/problems/merge-intervals/

analysis

Intercal problem can always be solved through greedy algorithm. Here we also use greedy algorithm to solve it. But we cannot use the ending time to sort the intervals again like classic interval schedule problem. Because in classic interval schedule problem, we need to accept as more intervals as possible. But here we need to get as less intervals as possible. So here we need to focus on the starting time as well and to the priority. So we first sort the intervals by their ascenging starting time. Then traverse the intervals, if we find a interval whose starting time is lower or equal to the current ending time, we need to update current ending time. If not, that means we can not merge it to the previous interval, so we just add it into result array and update starting time and ending time.

It leads to $O(n)$.

code

Greedy Algorithm $O(n)$

No.57 Insert Interval

https://leetcode.com/problems/insert-interval/

analysis

This problem is almost the same as No.56, we can just add the $newInterval$ to $intervals$, which can simplify the problem to No.56 throughly. Then we just follow the algorithm in No.56. The problem will be solved.

But what about thinking a different and interesting solution. That is realized without merging $intervals$ and $newInterval$ at first. We use a mark to memorize whether we have add $newInterval$ to the result array. During traversla, if there is a overlapping, we may merge the interval to $newInterval$. If not we will add $interval$ to the result array, as well as the $newInterval$ under the condition that $newInterval.end<interval.start$ and the newInterval has not been added before.

Greedy Algorithm $O(n)​$

No.252 Meeting Rooms

https://leetcode.com/problems/meeting-rooms/

analysis

This problem can be simply solved by greedy alglrithm. At first, we sort the intervals by their ending time. And then we traverse the intervals and update current time. Then we compare the start time of next interval to current time. If $interval.start>=cur$, there is a conflict.

It leads to $O(n)$.

Greedy Algorithm, $O(n)​$

No.253 Meeting Rooms II

analysis

This problem is similar to No.630. If we find a meeting which is in the condition that $-cur>interval.start​$, we need to again pop element from the heap untill we find the room that satisfying the condition that $-cur \le interval.start​$ and waste the least time.

This greedy algorithm leads to $O(n^{2})$.

No.435 Non-overlapping Intervals

https://leetcode.com/problems/non-overlapping-intervals/

analysis

This problem is almost the same as the classic interval schedule problem. We use greedy algorithm to solve it. At first, we sort the intervals by the ascending ending time. Then traverse the intervals, if there is a conflict, we need to delete it. If not, update current ending time. We can prove it is optimal by proving all the steps “stay ahead” and contradiction.

code

Greedy Algorithm $O(n)​$

No.452 Minimum Number of Arrows to Burst Balloons

https://leetcode.com/problems/minimum-number-of-arrows-to-burst-balloons/

analysis

We can make an analogy between this problem and No.253 Meeting Rooms II. No.253 needs to return the minumum length of the vertical diameter, however this problem wants a minimum length of horizental diameter, which is nuch simpler to obtain. The core thought is that if there exists conflict, we can use just one arrow shot to burst all conflict balloons. So we firstly sort the points of balloons. Then we traverse the balloons. If there is no conflict, we need one more arrow.

It leads to $O(n)$.

code

Greedy Algorithm $O(n)$
