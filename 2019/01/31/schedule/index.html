No.621 Task Schedule

https://leetcode.com/problems/task-scheduler/

analysis

We can solve thos problem based on observation and mathematics. For example, given $[‘A’, ‘A’, ‘B’, ‘B’, ‘C’, ‘C’, ‘C’ ]​$ and $2​$, it is obvious that the interval numbers should at least have a length of $(maxNum-1)(n+1)​$ before the last $C​$ so that we can schedule all the $C​$ tasks. And for the last $C​$, we just add one more interval to the result, then return. Then another example is $[‘A’, ‘A’, ‘B’, ‘B’, ‘B’, ‘C’, ‘C’, ‘C’ ]​$ and $2​$. The numbers of $B​$ and $C​$ are both the largest number, so we need to add two more intervals to the result. So we conclude that we can calculate the number of all the tasks occur the most frequently and add the number to the result.

Another important thing is a tricky example, such as $[‘A’, ‘A’, ‘B’, ‘B’, ‘C’, ‘C’]$ and 0. We may get $1$ following the previous algorithm, but actually we should return $6$. This occurs because the number of different tasks is more than $n+1$, so we don’t need to considering adding $idle$ interval.

Overall, return the maximum of $len(tasks)$ and the result from the previous algorithm should be reasonable. It leads to $O(n)$.

No.207 Course Schedule

https://leetcode.com/problems/course-schedule/

analysis

This is a typical graph problem. The prerequisites array forms a directed graph. Here we use breath-first-search to solve the problem. We at first establish a graph mapping the courses and their prerequisites. Then we also need to know how many times each course are taben to be a prerequisite for others. If a course which are not taken by any other courses to be a prerequisite, that means it’s free for us to take that course. For those classes which are not prerequisites, we will traverse them and update the courseNum simultaneously. During the traversal, we need to find their prerequisites and reduce the number by $1$ for each prerequisite. If the number is reduced to $0$, we can add this course to the deque, which means no course should take this course as prerequisite any more. At last, we need to campare $courseNum$ and $numCourses$ and return the result. It leads to $O(n)$.

BFS using deque, $O(len(prerequistes))​$ (the edge of graph)

No.630 Course Schedule III

https://leetcode.com/problems/course-schedule-iii/

analysis

This is a typical greedy alforithm problem, which is a little similar to interval scheduling. So similarly we need to sort the courses by their ending time. Then we traversal the courses, and everytime if we find that the accumulative starting time is higher than the end, we need to reduce the courses with the longest lasting time so that we can gain more time to add more new courses.

Here we use a heap to maintain a priority queue. In python, we can use $heapq$ to maintain a min heap. And we can transfer our min heap to ‘max heap’ using negative numbers. Everytime we traverse the courses, we push the nehative number of lasting time to the heap. As soon as we need to delete a course, we just pop the minmum element from the heap. Since we use negative numbers before, we can get the biggest positive lasting time.

It leads to $O(n)$.

This problem is also possible to be solved using DP.

code

Greedy Algorithm with priority queue (heap), $O(n^{2})​$

No.252 Meeting Rooms

https://leetcode.com/problems/meeting-rooms/

analysis

This problem can be simply solved by greedy alglrithm. At first, we sort the intervals by their ending time. And then we traverse the intervals and update current time. Then we compare the start time of next interval to current time. If $interval.start>=cur$, there is a conflict.

It leads to $O(n)$.
